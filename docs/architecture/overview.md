# AIryss vNext - アーキテクチャ概要

## システム全体像

AIryss vNext は「Human が AI エージェントを監督しながら開発を進める」ブラウザ IDE / オーケストレーション基盤です。
実装を担う Iryss と設計レビューを担う Aryss の出力を Human が統治・調整することで、AI の自律性と品質管理を両立させます。

```
[ブラウザ]
    │
    ▼
[nginx] ─────────────┐
    │                │
    ▼                ▼
[web (Next.js)]   [api (FastAPI)]
                      │
              ┌───────┼───────┐
              ▼       ▼       ▼
         [postgres] [redis] [workspace-runner]
                               │
                               ▼
                    [Docker ワークスペースコンテナ]
```

---

## エージェント責務境界

AIryss における中心的な概念は「Human を介した AI エージェント間の責務分離」です。

```
┌─────────────────────────────────────────────────────┐
│                      Human                          │
│  ・タスク指示 → Iryss                               │
│  ・レビュー依頼 → Aryss                             │
│  ・両エージェントの出力を評価し最終決定              │
└───────────────┬─────────────────┬───────────────────┘
                │                 │
                ▼                 ▼
         ┌──────────┐      ┌──────────┐
         │  Iryss   │      │  Aryss   │
         │ 実装      │      │ 設計     │
         │ エージェント│      │ レビュー │
         │          │      │ エージェント│
         └──────────┘      └──────────┘
              ↑ 成果物をレビューに渡す（Human 経由）
```

| エージェント | 役割 | やらないこと |
|-------------|------|------------|
| Iryss | コーディング・リファクタリング・テスト作成 | 設計判断・レビュー |
| Aryss | 設計評価・品質レビュー・代替案提示 | 実装・ファイル変更 |
| Human | 指示・承認・最終意思決定 | 細かい実装作業 |
| AIryss | 上記全員の協働を支えるプラットフォーム | エージェント自身の判断 |

## インフラ構成（コンテナ）

### nginx
- リバースプロキシ
- `/` → web、`/api` → api にルーティング
- TLS 終端（本番環境）

### web (Next.js)
- ブラウザ IDE UI
- Monaco Editor / xterm.js 統合予定
- WebSocket でターミナル・LSP と通信

### api (FastAPI)
- REST / WebSocket API
- 認証・認可
- ワークスペースのライフサイクル管理
- Aryss AI エージェントとのブリッジ

### postgres
- ユーザー・プロジェクト・ワークスペースのメタデータ保存

### redis
- セッション管理
- ワーカーへのジョブキュー（RQ / Celery 予定）
- リアルタイムイベントの Pub/Sub

### workspace-runner (Python ワーカー)
- Redis キューからジョブを受信
- Docker API を使ってワークスペースコンテナを起動・停止
- コンテナへのファイル操作・ターミナルセッション管理

---

## セキュリティ方針

- 各ワークスペースは独立したコンテナで実行（プロセス分離）
- ネットワークレベルでの分離（Docker network）
- API は JWT 認証
- ワークスペースコンテナへの直接アクセスは api 経由のみ
